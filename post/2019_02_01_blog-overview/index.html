This is a brief illustration of how to build this site.

Overview

This website uses hugo to generate static web pages, and hosting on github page.

Why hugo ?

Pros

  • hugo is one of the most popular framework
  • using newest lang - Go
  • light-weighted
  • can build at local
  • Fast loading/generating page

Cons

  • only static page, not dynamic (can’t use something related to database)
  • doesn’t support nested category


Why Github page ?

Pros

  • Free
  • Fast
  • Easy to use
  • Insurance - data both at github and local

Cons

  • only support static page
  • everything is explode on github


Start building page

Steps:
1. setup hugo at local
2. create git-repo
3. push on git

Setup

Hugo

official quick start

  • Install hugo

    1

    brew install hugo

  • Create a new site

    1

    hugo new site <your-site>

  • Add a theme

    pick theme here: official theme
    and follow the instuction provided

  • config

    in config.toml
    baseURL = “https://your-git-username.github.io/"

  • Add a post

    1
2
3

    # note: some themes may have different path
# i.e. hugo new post/...
hugo new posts/my-first-post.md

  • start a hugo server loacally

    1

    hugo server -D

Github

  • create git repo

    github official guide
    Head over to GitHub and create a new repository named username.github.io, where username is your username (or organization name) on GitHub.

    #!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public
# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

# Come Back up to the Project Root
cd ..
